\begin{abstract}
\large
Network Functions Virtualisation (NFV) decouples network functions from
proprietary hardware, allowing them to run as software on commodity
infrastructure. This project designs and implements a \textbf{mini NFVI
(NFV Infrastructure) stack} that emulates the ETSI NFV architectural
framework at small scale, using cloud-native technologies instead of a
traditional OpenStack Virtual Infrastructure Manager (VIM). A three-node
Kubernetes cluster is provisioned with Infrastructure-as-Code (Vagrant and
Ansible) on a single 16~GB host, running the lightweight k3s distribution.
A Cloud-Native Network Function (CNF) is deployed with \textbf{multi-NIC
networking} (Multus~+~macvlan~+~Whereabouts IPAM) to separate the
management, control, and user (data) planes following telco best practice.
The platform integrates an observability stack (Prometheus~+~Grafana) and
horizontal auto-scaling (HPA). Five test cases validate multi-NIC
connectivity, data-plane throughput, service load balancing, auto-scaling,
and observability. The work demonstrates a complete, reproducible mapping
from the ETSI NFV reference model onto Kubernetes primitives.
\end{abstract}
